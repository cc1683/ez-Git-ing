What is Git?

Git is a version control system for tracking changes in computer files and coordinating work on those files among multiple people. --Wikipedia

Installation

For Linux user

sudo apt-get install git-all

For Mac and Window user

Can directly download from here

Configure Git

Set the git username and email (globally)

git config -g user.name username
git config -g user.email user@gmail.com

If dont want to set globally just remove the -g in the command

Check your git username and email

git config user.name
git config user.email
git config -l

Create a repository

Initiate an empty git repository

git init

Add all the modified file into staging area

git add .

Add only specific file into staging area

git add filename

Add all the html file

git add *.html

Add the whole directory into staging area

git add foldername

Add the specific file within a directory

git add foldername/filename

Remove a file from staging area

git rm --cached filename

Remove directory from staging area

git rm --cached -r foldername

Make commit

Commit the changes

git commit -m "commit messages"

Show the commit history

git log
git log --oneline

Show what does a commit change

git show commitnumber

Undoing things

checkout, revert, reset are use to undoing change in git. However, they play different role depand on the conditions.

Check back the existing commit (git will automatic create an another branch for you to review the previous commit)

git checkout commitnumber

Revert back to any previous commit

git revert commitnumber

Reset the commit (danger). After reset, the changes will still stay in staging area. If want to hard reset, add --hard in the code below

git reset commitnumber

Branching

Create a new branch

git branch branchname

Show branch

git branch -a

Switch to another branch

git checkout branchname

Create a new branch and switch to it

git checkout -b branchname

Merge branch (Merge branch to you current stay branch)

git merge branchname
